Codename: Driftpacito

Codename: Driftpacito is an addictive mobile game made in Unity that was based around the idea that the only input the player has is touch. There are no buttons (apart from the UI) and no controls other than simply touching the screen. This was heavily influenced by Flappy Bird, a highly addictive game that also only had a single touch input.

The player character moves to from side to side on its own, and touching and holding the screen allows the player to stop this movement. The goal of the game is to dodge debris in an ever speeding-up game, all in an effort to set a high score. The player is also given the option of collecting small meteors which offer different abilities based on colour. These abilities include slowing the game down whilst keeping the score multiplier; increasing the score multiplier and an overshield which stops the player from losing should they collide with a piece of debris.

A beta version of this game is available to play on Android. If you'd like to give it a shot, then get in touch below :).

AI Flocking Behaviours in C++

A university project in which we were tasked to create a component system based AI with flocking behaviours, which include allignment, separation and cohesion.

These three individual behaviours, along with a wander multiplier, can be altered in real-time using ImGui. ImGui also allows the user to add more entities, known as Pinkies and Imps (yes, the enemies from Doom).

The respective Pinkies and Imps use Axis Aligned Bounding Boxes (AABB) to avoid the walls around the square in which they can walk. They use this same collision detection to avoid a box that can be placed randomly in the scene by the user using ImGui.

The entities don't collide with eachother, which means that they will all merge together when allignment and cohesion are turned all the way up and the separation value is near zero.

The entities choose which direction to wander in by choosing a random poin on a circle in front of them and drawing a line from the center to this point. This can be seen here.

This project was made within a framework given to us by the University, meaning we didn't have to start from scratch.

GLSL Shaders with C++

One of the final assignments that was given at university was to implement different shaders in a scene that was given to us. The initial scene contained one directional light using the Phong lighting model.

The first task was to add shadows to the scene based off this directional light and then implement PFC and Stratified Poisson Sampling to eliminate shadow acne.

The next task was to add two more lights: a point light and a spotlight.

The spotlight has an adjustable inner and outer cone, which can be used to give the edge of the light different amounts of softness.

All of the different values of each light, including RGB values, are adjustable in real-time using ImGui.
